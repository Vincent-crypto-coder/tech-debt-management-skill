## 类别说明

| 类别 | 描述 | 示例 |
|------|------|------|
| 代码债务 | 代码质量问题 | 重复代码、过长函数、命名不佳 |
| 架构债务 | 设计问题 | 模块耦合、分层混乱、God Class |
| 测试债务 | 测试不足 | 覆盖率低、测试脆弱、缺少集成测试 |
| 文档债务 | 文档缺失 | 文档过时、缺失、不准确 |
| 依赖债务 | 依赖问题 | 版本过时、安全漏洞、许可证风险 |
| 基础设施债务 | 运维问题 | 手动部署、监控缺失、配置混乱 |

## 优先级计算公式

```
优先级 = (频率 × 范围 × 风险) / 成本
```

阈值：
- 高优先级：>= 15
- 中优先级：8-14
- 低优先级：< 8
